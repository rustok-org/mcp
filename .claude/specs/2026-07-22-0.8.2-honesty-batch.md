# PR — 0.8.2: батч честности установщика и шима (итог живой пробы 0.8.1)

> Статус: **СПЕКА, Гейт-1.** База: `origin/main = 4701197` (после #85), тег `wallet-tui-v0.8.1` = `d8f6639`.
> Кресло: Инженер. Ратифицировано Капитаном: **вариант (д)** — №1 (утечка RPC в `inspect`) закрывается
> в 0.8.2 **только текстом**, код-половина отложена на следующий релиз.
> Источники находок: живая проба 0.8.1 свежей сессией («тестировщик») + два read-only аудита по `main`.

## Цель — один абзац

0.8.1 снял стену cosign, и живая установка это подтвердила: одна команда, `EXIT 0`, без вмешательства.
Но проба и аудиты вскрыли один общий класс дефектов: **мы утверждаем то, чего не делаем**. Установщик
говорит «cosign не установлен» про битый файл; рапортует провал, уже установив рабочий шим; учит
несуществующему тегу. Шим обещает в `--help` проверку leftovers, которой нет, и защиту от `podman
inspect`, которой на до-`connect` пути нет; `uninstall` печатает «stopped» и «removed», не проверив.
Это та же линия, которую мы весь день чистили с четырёх витрин — здесь она внутри продукта.
0.8.2 не добавляет возможностей: она приводит слова в соответствие делам.

## Скоуп PR-1 — `scripts/install.sh`

1. **Детект cosign — 2 состояния вместо 3** (рекомендация Ревьюера, принята):
   ```sh
   cosign_state() {
       cosign version >/dev/null 2>&1 && { echo works; return 0; }
       echo unavailable
   }
   ```
   `works` → `verify` → `die` при провале (**ветку не трогаем**, fail-closed цел).
   `unavailable` → один `skip_provenance`, текст несёт **обе** причины: не найден в PATH ЛИБО найден,
   но не запускается (нет `+x` · чужая архитектура · обрезанная закачка).
   **Почему 2, а не 3:** граница `126↔127` не существует. Замерено на файлах с `+x`: случайный
   бинарный мусор → **2**, обрезанный ELF → **126**, «почти текст» → **127**. Ключ «127 = absent»
   назвал бы часть битых файлов «не установлен» — ровно та ложь, которую чиним. `command -v`
   уходит совсем: он отвечает на другой вопрос и по-разному в разных оболочках.
2. **Три файловые записи получают обработку ошибок** (аудит, HIGH):
   - `mkdir -p "$INSTALL_DIR"` (`:181`) → `die` с нашим префиксом. Фатально, но честно.
   - `chmod +x "$shim_tmp"` (`:187`) → `die` **с уборкой** `$shim_tmp` (сейчас мусор остаётся —
     ровно то, ради чего блок написан через временный файл и атомарный `mv`).
   - **запись в профиль** (`:120-127`) → **НЕ фатально**. Шим к этому моменту установлен и работает;
     ронять установку из-за нетронутого dotfile — врать о результате. Деградируем до уже
     существующего в скрипте поведения `RUSTOK_NO_MODIFY_PATH`: предупредить, напечатать строку для
     ручного добавления, выйти **0**.
3. **Гейт правки профиля — по грепу самого профиля**, не по `already_on_path` сессии (оговорка
   Ревьюера). Иначе сломаем легитимный случай: путь есть в текущей сессии от родителя, но в новом
   шелле пропадёт, и человек останется без `rustok`.
4. **Замена существующего `rustok` объявляется строкой** в выводе (проба: старый симлинк затёрся молча).
5. **Шапка скрипта `:4-5`** — `mcp/vX.Y.Z/` → `mcp/wallet-tui-vX.Y.Z/`. Доказано: `v0.8.1` → **404**,
   `wallet-tui-v0.8.1` → **200**. Строку отката `:198` починили в 0.8.1, шапку пропустили.
6. **`docs/TROUBLESHOOTING.md` в тексте отказа → абсолютный URL.** Сообщение срабатывает в
   `curl | sh`, где ни клона, ни каталога `docs/` у человека нет.
7. **fish / csh** (`profile_file()`): не писать `export PATH=…` в файл, который эта оболочка не читает.
   Для них — печатать инструкцию под их синтаксис, профиль не трогать. Сейчас мы пишем в `.profile`
   и утверждаем «open a new shell» — для fish это заведомая неправда.
8. **Незаданный `$HOME`** — наш префикс вместо сырой ошибки оболочки.

## Скоуп PR-2 — `cli/rustok`

1. **№1-текст. `--help` описывает три режима честно** (ратифицировано (д)):
   - после `connect` (секрет есть, podman) — значение вне argv, вне конфигов, **вне `inspect`**;
   - до `connect` / голый `RUSTOK_RPC_URLS_*` на podman — name-forward: вне argv и истории, **но
     видно в `podman inspect`**;
   - docker-fallback — литеральный `-e`, второй эшелон, **видно в `inspect`**.
2. **№1-текст, вторая половина: warn-note при name-forward** — оба движка, **до exec**, с тестом.
   ⚠️ **Ловушка:** `wallet_run_args` отдаёт argv **через stdout**; `say` внутри неё превратит
   предупреждение в аргументы `podman run`. Warn-note пишется в **stderr** (либо печатается
   вызывающим до exec). Тест обязан проверять и текст предупреждения, **и** что argv не изменился.
3. **№2. `uninstall --purge-keys`: провал `volume rm` больше не молчит.** Сейчас `volume rm … && echo`
   при провале не печатает ничего нашего и выходит `0`. Единственная команда, чья работа — сказать
   «ключи уничтожены», обязана отличать полный успех от частичного: именованная строка о каждом
   невыжатом томе и **ненулевой** код возврата.
4. **№3. `uninstall` не рапортует «stopped N», не проверив.** Сейчас `stop … || true` + безусловное
   сообщение. Считать реально остановленные; о неостановленных сказать. Отдельная `rustok stop` то же
   самое делает правильно — приводим к её поведению.
5. **№5. Тот же паттерн на удалении файлов паролей** (`rm -f … || true` + безусловное «removed»).
6. **№4. Кавычки в docker-fallback** (`:201`) — единственное из четырёх мест, где путь к файлу пароля
   вставлен без кавычек. `$HOME` с пробелом рвёт argv; хуже всего в `connect`, который вмораживает
   испорченный argv в конфиг MCP-клиента навсегда.
7. **`doctor` становится read-only** — сейчас пишет `~/.config/rustok/config`. Диагностику запускают
   смотреть, не менять.
8. **`doctor`/`status` различают «кошелёк не создан» и «создан, но не запущен»** и ведут к `rustok init`.
   Сейчас у свежеустановившегося всё зелёное и ни одной подсказки, что делать дальше.
9. **`--help` перестаёт обещать проверку leftovers**, которой нет (либо проверка появляется; выбираем
   первое — минимум скоупа).
10. **Ось шима `VERSION="0.3.0"` → `0.3.1`** — команды шима меняют поведение. На этой оси гварда нет,
    поэтому пункт явный.

## Отложено — НЕ входит в 0.8.2

- **№1-код: закрыть саму утечку** — на podman создавать секрет для `RUSTOK_RPC_URLS_*` на лету (как
  это делает `connect`) и подавать через `--secret`, вместо name-forward. Docker секретов не имеет —
  там утечка неустранима и остаётся объявленной в тексте.
  **Тест-место: `tests/shim/run-tests.sh:433-445`** — существующий тест «RPC by NAME only» проверяет
  отсутствие значения в argv-логе, что НЕ равно гарантии про `inspect`; расширять надо там.
  **`/security-review` на диф обязателен** (требование Ревьюера, Гейт-1). В 0.8.2 не входит по
  ратификации Капитана (вариант «д»).
- Находки параллельной пробы тестировщика по живой 0.8.1 — сводим отдельно, в этот батч не вмешиваются.

## Тест-план

**Правило №0 — каждый новый тест доказан красным до фикса.**

**Install-сюита (`tests/install/run-tests.sh`), всё под `sh` — по настоящему пути доставки:**
- **кейс B: cosign есть, `+x` нет → под `sh`** (сейчас невозможен: `command -v` уводит в absent).
  Инвариант: НЕ verify-ветка И текст упоминает «не запускается», а не только «не установлен».
- **кейс C: cosign с `+x`, но не исполняется** — три подварианта (мусор → `2`, обрезанный ELF → `126`,
  «почти текст» → `127`), все три обязаны дать одну ветку `unavailable`.
- сохранить: works+pass, works+fail→die (fail-closed), оба текста-инварианта.
- **новые кейсы отказов ФС**: `mkdir` падает → die с префиксом, ничего не установлено; `chmod` падает
  → die с префиксом **и `$shim_tmp` убран**; **запись в профиль падает → exit 0, шим на месте,
  напечатана ручная строка PATH**.
- гейт профиля: путь уже в профиле → блок не дописан; путь в сессии, но не в профиле → дописан.

**Shim-сюита (`tests/shim/run-tests.sh`):**
- warn-note при name-forward: текст присутствует **и argv побайтово не изменился** (ловушка stdout);
- `volume rm` падает → именованная строка + ненулевой код;
- `stop` падает → не рапортуем «stopped N», называем неостановленные;
- `$HOME`/`XDG_CONFIG_HOME` с пробелом → argv docker-fallback остаётся одним токеном;
- `doctor` не создаёт `~/.config/rustok/config` (проверка отсутствия файла после прогона);
- `doctor`/`status` на машине без кошелька ведут к `init`.

**Мутации-контроли:** снять кавычки обратно → тест с пробелом краснеет; вернуть `|| true` → тест
`stop` краснеет; вернуть `&& echo` → тест `volume rm` краснеет.

**Полный CI-свип локально (9 джоб)** — `shellcheck`/`hadolint` контейнерами, `docker-smoke` под podman
с честной пометкой о замещении движка.

## Критерии приёмки

1. Ни одна строка `--help` и ни одно сообщение не обещает того, чего код не делает — проверено
   построчно по обоим файлам, а не выборочно.
2. Ни одна деструктивная команда (`uninstall`, `--purge-keys`) не рапортует успех при частичном провале.
3. Установка не роняется из-за того, что уже успешно сделано (профиль), и не оставляет мусора при
   том, что не сделано (`$shim_tmp`).
4. Детект cosign не зависит от оболочки; все три подварианта битого файла дают одну ветку.
5. 6 точек версии = `0.8.2`, ось шима = `0.3.1`, URL-тег синхронен грепом.
6. Мани-инвариант цел: установщик не касается секрета, пароль кейринга не попадает в argv/env-file/конфиг.
