## Вердикт

Продукт **работает** end-to-end. Причина «800 скачиваний / 0 звёзд» — не «не могут установить», а **протокольный баг в замороженной линии: строгие MCP-клиенты (Claude Code ≥2.1) не могут завершить handshake**. Целевая аудитория агентского кошелька — агент-клиенты, и продукт с ними не разговаривает.

## Блокеры воронки (по убыванию)

### 🔴 B1: протокольный баг `"error": null` — строгие клиенты не коннектятся

Каждый ответ замороженной линии содержит `"error": null` рядом с `result` (подтверждено в живом ответе initialize). Это ровно баг, из-за которого Claude Code 2.1.2 молча отвергает handshake (30s таймаут, «нет кошелька») — найден первым реальным пользователем 2026-07-15, исправлен в 0.6.0 (`to_wire` exclude), **но frozen-линия фикс не получила и не получит никогда без бэкпорта**. Для продукта «кошелёк для агентов» это фатальная воронка: агент-клиент не может даже поздороваться.

**Фикс:** бэкпорт `to_wire` XOR-фикса из main на тег `wallet-v0.4.0` → релиз `wallet-v0.4.1` + обновить `latest` (заморозка на v0.4.0 была из-за поведения execute_send, патч протокола его не меняет). Часы работы.

### 🟡 B2: установка — 6 ручных шагов, не «одна команда»

Скилл ведёт: read пароля → docker run create-wallet → запись env-file 0600 → правка MCP-конфига с абсолютным путём. Плюс env-file метод, который сам проект депрекейтнул. У TUI есть install.sh — у автономного нет.

**Фикс:** one-command installer для автономного продукта (по образцу TUI-шима, минимальная версия) или переписанный скилл с одной командой онбординга.

### 🟡 B3: обязательный RPC URL

Превью/балансы требуют RPC (Alchemy-ключ = регистрация). Публичный RPC работает (доказано), но это шаг настройки.

**Фикс:** дефолт на публичный RPC из коробки (публичные эндпоинты как emergency-дефолт с пометкой) — снимает обязательный шаг.

### 🟡 B4: frozen-линия без канала фиксов

Следующая ломка протокола/клиента снова убьёт продукт. Решается inc2 (unified autonomous) в спокойном темпе после починки воронки.

### 🟡 B5: amd64-only (вероятно)

Apple Silicon/ARM-пользователи падают на pull/run. Проверить наличие arm64-манифеста при релизе 0.4.1.

## Что НЕ является блокером (проверено)

Приватность образа, заморозка latest, онбординг headless, работа инструментов, публичный RPC, честность ошибок.

## План (предлагаю Капитану)

1. **Бэкпорт XOR-фикса → wallet-v0.4.1 + latest** (B1) — главный разблокировщик.
2. **One-command install для автономного** (B2) + RPC-дефолт (B3).
3. Проверка arm64 при релизе (B5).
4. Дальше по плану: inc2 (autonomous на unified-ядре, с security review) → inc3 → хвосты.
